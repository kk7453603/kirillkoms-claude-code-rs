```markdown
## Usage Scenarios

Practical examples of what Claude Code RS can do. Start the agent with `./target/release/claude-code` and type the prompt shown.

### 1. Рефакторинг функции

**Задача:** Функция стала слишком большой и делает несколько вещей сразу.

```
> Прочти crates/cc-api/src/providers/openai_translate.rs и разбей функцию
  translate_request на более мелкие, каждая с одной ответственностью.
  Не меняй поведение, только структуру.
```

Агент читает файл, выделяет логические части (system messages, tools, tool_choice),
создаёт вспомогательные функции и обновляет файл. Предлагает запустить тесты для проверки.

### 2. Code Review перед PR

**Задача:** Проверить изменения в текущей ветке перед тем как создавать PR.

```
/review
```

Агент запускает `git diff main`, анализирует изменения на предмет ошибок,
нарушений стиля и потенциальных проблем. Возвращает структурированный список замечаний
с указанием файлов и строк.

### 3. Дебаггинг по стектрейсу

**Задача:** Приложение падает с паникой, нужно найти и исправить причину.

```
> Вот стектрейс:
  thread 'main' panicked at 'index out of bounds: the len is 3 but the index is 5',
  crates/cc-api/src/streaming.rs:42
  Найди причину и исправь.
```

Агент читает `streaming.rs` вокруг строки 42, анализирует логику, находит условие
при котором индекс выходит за границы, исправляет код и предлагает добавить тест
на этот edge case.

### 4. Добавление unit-тестов

**Задача:** Модуль написан без тестов, нужно покрыть основные случаи.

```
> Изучи crates/cc-api/src/providers/openai_translate.rs и напиши unit-тесты
  для функции translate_response. Покрой happy path и edge cases (пустой контент,
  несколько tool calls, отсутствующий role).
```

Агент читает существующие тесты в файле для понимания стиля, пишет новые тесты
в блок `#[cfg(test)]` в конце файла и запускает `cargo test -p cc-api`.

### 5. Работа с git

**Задача:** Закончили фичу, нужно закоммитить и создать PR.

```
/commit
```

Агент запускает `git diff --staged`, анализирует изменения и предлагает
commit message в формате conventional commits. После подтверждения коммитит.

```
/commit-push-pr
```

Комбинированная команда: коммит → push → создание PR с автоматически
сгенерированным описанием на основе diff.

### 6. Поиск по кодовой базе

**Задача:** Понять как устроена определённая часть системы.

```
> Найди все места где создаётся ApiError::Timeout и объясни в каких ситуациях
  он возникает. Покажи путь от вызова до обработки ошибки.
```

Агент использует Grep для поиска всех вхождений, читает контекст вокруг каждого,
строит объяснение потока ошибки от источника до обработчика.

### 7. Многоагентная задача

**Задача:** Одновременно проверить несколько независимых частей кодовой базы.

```
> Запусти три параллельных агента:
  1. Проверь наличие документации на публичных функциях в crates/cc-api/src/
  2. Найди все unwrap() в crates/cc-tools/src/ которые могут паниковать
  3. Проверь что все env vars из cc-config/src/env.rs упомянуты в README.md
```

Агент использует инструмент Agent для запуска трёх субагентов параллельно.
Каждый возвращает отчёт, основной агент объединяет результаты.

```
